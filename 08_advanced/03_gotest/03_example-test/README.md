# 示例测试 (Example Test)

演示代码用法 + 自动验证输出。这种测试兼具文档 + 测试双重作用，适合展示函数的使用方式。

> 可以生成文档，也可以验证代码逻辑

## 函数格式

```
func ExampleXxx() {
    // 调用要演示的函数
    fmt.Println(Add(2, 3))
    // Output: 5
}
```

- 函数名必须以 Example 开头并且没有参数
- 给用户提供代码示例，在注释中写出预期输出
- 必须有 `// Output: 注释`（区分大小写）
- 测试框架会运行函数并比较实际输出和 Output: 内容是否一致
- Go 测试框架会捕获 stdout 并对比是否与注释中的输出一致
- 多次调用测试 Output 也需要多行输出

## 命名规则

| 函数命名             | 含义                        |
|------------------|---------------------------|
| `Example()`      | 包级示例                      |
| `ExampleXxx()`   | 针对某个函数/类型的示例              |
| `ExampleXxx_y()` | 针对函数或方法的特定用例（`_` 后可自定义名字） |


## 文档生成作用

运行 `go doc -all` 或生成在线文档时（如 pkg.go.dev），示例函数会自动出现在文档里，变成可运行的示例代码

## 注意事项

| 限制                         | 说明                      |
|----------------------------|-------------------------|
| 函数不能有参数                    | 必须是 `func ExampleXxx()` |
| 无法使用 `t *testing.T`        | 不能调用 `t.Error()` 等函数    |
| 输出严格匹配                     | 空格、换行都要一致               |
| 默认比较标准输出                   | 不比较返回值或日志               |
| 可以用 `// Unordered output:` | 表示输出顺序不要求一致（Go 1.15+）   |
