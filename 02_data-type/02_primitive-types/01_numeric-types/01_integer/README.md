## 整数类型（Integer Types）

整数类型是 Go 中最常用的数值类型，分为两类：有符号整数与无符号整数

### 有符号整数（Signed Integers）

| 类型      | 位数           | 取值范围（约）                        |
|---------|--------------|--------------------------------|
| `int8`  | 8            | -128 ~ 127                     |
| `int16` | 16           | -32,768 ~ 32,767               |
| `int32` | 32           | -2,147,483,648 ~ 2,147,483,647 |
| `int64` | 64           | -9.22e18 ~ 9.22e18             |
| `int`   | 平台相关（32或64位） | 取决于系统架构                        |

### 无符号整数（Unsigned Integers）

| 类型       | 位数           | 取值范围（约）           |
|----------|--------------|-------------------|
| `uint8`  | 8            | 0 ~ 255           |
| `uint16` | 16           | 0 ~ 65,535        |
| `uint32` | 32           | 0 ~ 4,294,967,295 |
| `uint64` | 64           | 0 ~ 1.84e19       |
| `uint`   | 平台相关（32或64位） | 同样取决于架构           |

> `int` 和 `uint` 的位宽依赖 CPU 架构（32/64 位系统），通常推荐使用 `int`

### 特殊整数别名

byte 和 rune。它们都与字符（character）处理相关，但用途和底层含义不同

|   类型   |  本质类型   | 含义                              | 字节长度 | 常见用途                          |
|:------:|:-------:|:--------------------------------|:----:|:------------------------------|
| `byte` | `uint8` | 表示一个 **字节（8 位）**                | 1 字节 | 处理 **原始二进制数据** 或 **ASCII 字符** |
| `rune` | `int32` | 表示一个 **Unicode 码点（code point）** | 4 字节 | 处理 **Unicode 字符（如中文、emoji）**  |

- byte -> 一个字节（0~255）
- rune -> 一个 Unicode 字符

### 整数的特点与操作

#### 类型安全

不同整数类型不能直接混用，必须显式转换：

```go
var a int = 10
var b int8 = 3
// var c = a + b    //  编译错误：类型不匹配
var c = a + int(b) //  显式转换
```

#### 算术运算

支持：

* 加法：`+`
* 减法：`-`
* 乘法：`*`
* 除法：`/`
* 取模：`%`

```go
x := 10
y := 3
fmt.Println(x / y) // 输出 3（整数除法，结果为整型）
```

> 整数除法会截断小数部分，不会四舍五入

#### 自增与自减

```go
i := 5
i++ // 等价于 i = i + 1
i-- // 等价于 i = i - 1
```

> 注意：`i++` 不是表达式，不能出现在赋值语句中，如 `j = i++`

### 溢出（Overflow）

当运算结果超过类型范围时，会回绕（wrap around）

```go
var x uint8 = 255
x++ // 溢出 -> 0
```

> Go 不会自动检测整数溢出，编译器不会报错
