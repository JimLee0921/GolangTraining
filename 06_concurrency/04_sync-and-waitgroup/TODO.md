Go并发的核心基础原语包括 Goroutine、Channel 和 sync 包。
Goroutine是轻量级线程，Channel用于Goroutine之间的通信，
sync包则提供了互斥锁（Mutex）、读写锁（RWMutex）和原子操作等工具来同步和保护共享资源。

| 分类    | 原语                                 | 作用简述                       |
|-------|------------------------------------|----------------------------|
| 基础锁   | `Mutex` / `RWMutex`                | 互斥访问，读写分离                  |
| 一次性执行 | `Once`                             | 保证某函数只执行一次（并发安全）           |
| 条件变量  | `Cond`                             | 等待/通知机制（类似传统操作系统的条件变量）     |
| 等待组   | `WaitGroup`                        | 等待一组 goroutine 完成（常用于任务同步） |
| 单次通知  | `Pool`                             | 对象复用池，减少内存分配（性能优化）         |
| 临时存储  | `Map`                              | 并发安全 map（无须加锁）             |
| 原子操作  | `sync/atomic`                      | 无锁安全操作单变量                  |
| 其他封装  | `OnceValue` / `OnceFunc`（Go 1.21+） | Once 的泛型改进版本               |
