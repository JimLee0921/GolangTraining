## Err

`Scanner.Err()` 是 Scanner 的状态检查方法，
主要判断在 `Scan()` 返回 false 时，是正常结束（EOF）还是发生了错误

### 方法签名

```
func (s *Scanner) Err() error
```

### 功能说明

`Err()` 返回扫描过程中遇到的第一个非 EOF 错误。

- 如果扫描正常结束（EOF） -> 返回 nil
- 如果扫描发生错误 -> 返回错误对象（error）

> Scan() 返回 false 可能是正常结束也可能是出错，只有调用 Err() 才能区分

| 状态         | `Scan()` 返回 | `Err()` 返回 | 说明           |
|------------|-------------|------------|--------------|
| 正常扫描中      | `true`      | `nil`      | 有新 token     |
| 正常结束 (EOF) | `false`     | `nil`      | 到文件末尾        |
| 出错         | `false`     | 非 `nil`    | 出错或 token 过长 |

### 返回情况

| 场景               | 说明                      |
|------------------|-------------------------|
| I/O 错误           | 文件损坏、设备断开、权限错误          |
| 缓冲区不足            | 默认最大 token 长度是 **64KB** |
| 自定义 SplitFunc 出错 | 你的 SplitFunc 返回了错误      |
| 正常 EOF           | 返回 `nil`（非错误）           |

> 在自定义 SplitFunc 抛出错误时 Err 会捕获自定义分割逻辑中的异常
