# Go 空白标识符 `_`

## 1. 基本概念

* `_` 叫 **空白标识符 (blank identifier)**。
* 它可以出现在任何需要变量名的地方，但不会真正创建变量。
* 编译器会丢弃赋给 `_` 的值。
* 常用于 **丢弃不需要的值**，避免编译错误。

---

## 2. 常见用法

### 忽略函数返回值

```go
value, _ := someFunc()
```

* 如果函数返回多个值，但你只关心一部分，可以用 `_` 丢弃。

### 避免 “declared and not used” 错误

```go
_, err := someFunc()
```

* Go 不允许定义变量后不用，用 `_` 可以占位。

### `for range` 循环中丢弃索引或值

```go
nums := []int{1, 2, 3}
for _, v := range nums {
fmt.Println(v) // 丢弃索引
}
```

### 占位导入包（只执行 `init`）

```go
import _ "net/http/pprof"
```

* 导入包只为执行其 `init()`，而不直接使用包里的符号。

### 丢弃接口实现检查结果

```go
var _ io.Writer = (*MyWriter)(nil)
```

* 常用于 **编译期接口实现检查**。
* 这里不会真正赋值，只是用 `_` 来验证 `*MyWriter` 是否实现了 `io.Writer`。

---

## 注意事项

* `_` 不是一个变量，不能取地址 (`&`)，也不能读取。
* 每次用 `_` 都是一个新的“丢弃位置”，不会共享。
* 常量、值赋给 `_` 后，会被编译器认为“未使用”，直接丢掉。

---

## 总结

* `_` 是 Go 提供的“**黑洞变量**”。
* 用途：丢弃返回值、忽略循环变量、占位导入包、接口实现检查。
* 核心作用：**避免未使用错误，简化代码**。
